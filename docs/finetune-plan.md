# 火山方舟模型精调计划：AIGC 改写专用模型

## 一、目标

微调一个专用的"去AI味改写"模型，使其输出的文本能通过外部 AIGC 检测器（朱雀等），替换当前 `aigc-detect.ts` 中基于通用 LLM + Prompt 的改写方案。

### 核心指标

| 指标 | 当前 Prompt 改写 | 微调后目标 |
|------|-----------------|-----------|
| 困惑度检测 AI 评分 | 改写后 50-70 分 | < 35 分 |
| 改写幅度（bigram 变化率） | 15-25% | > 25% |
| 内容保真度（数字锚点保留） | > 80% | > 85% |
| 外部检测器（朱雀）通过率 | ~30% | > 70% |

---

## 二、技术方案选型

### 2.1 训练平台：火山方舟

- 控制台地址：https://console.volcengine.com/ark
- 优势：无需自建训练环境，训完直接 API 调用，费用透明

### 2.2 基座模型选择

| 候选模型 | 适合场景 | 预估费用 |
|---------|---------|---------|
| **Doubao-Seed-1.6-flash** | 轻量风格任务，速度快，推荐首选 | ~¥50-100/次训练 |
| Doubao-Seed-1.6 | 更复杂的改写理解 | ~¥200-500/次训练 |
| Doubao-1.5-pro-32k | 长文本改写（章节级） | 较贵 |

**推荐**：先用 **Doubao-Seed-1.6-flash** 做第一轮 SFT 验证效果，费用低、训练快。

### 2.3 训练路线（三阶段）

```
阶段 1: SFT 有监督微调（必须）
  ↓ 验证基础效果
阶段 2: DPO 直接偏好优化（推荐）
  ↓ 验证检测分数下降
阶段 3: 持续迭代（按需）
```

## 三、阶段 1：SFT 有监督微调

### 3.1 数据策略

**核心思路**：从真实人类小说中提取片段作为 target（改写目标），用 LLM 将其"AI 化"作为 input（待改写文本），训练模型学习 `AI风格 → 人类风格` 的映射。

#### 数据来源

| 来源 | 用途 | 数量 |
|------|------|------|
| `docs/凡人修仙传.txt` (144K行) | 人类写作 target | ~3000 片段 |
| `docs/我在精神病院学斩神.txt` (252K行) | 人类写作 target | ~3000 片段 |
| `docs/experiments/` 中的 AI 样本 | 直接用作 AI 风格参考 | ~20 篇 |

#### 数据构造方法

每条训练数据由 3 步构造：

1. **采样**：从人类小说中随机采样 200-500 字的段落（含叙述+对话混合片段）
2. **AI 化**：调用通用 LLM 将人类片段改写为"典型 AI 风格"（增加模板连接词、均匀化句长、使用高频动作描写）
3. **配对**：`(AI化文本, 原始人类文本)` 构成一条训练样本

### 3.2 数据格式（火山方舟 JSONL）

每行一个 JSON 对象，格式如下：

```jsonl
{"messages":[{"role":"system","content":"你是专业的文本润色编辑。将AI风格的文本改写为自然的人类写作风格，保持剧情和角色不变，只输出改写后的正文。"},{"role":"user","content":"[AI风格文本]"},{"role":"assistant","content":"[人类原始文本]"}]}
```

### 3.3 数据规模

| 数据集 | 条数 | 用途 |
|-------|------|------|
| 训练集 | 4000-5000 条 | 模型训练 |
| 验证集 | 500-600 条 | 监控过拟合 |
| **总计** | **~5500 条** | |

> 火山方舟支持自动 80/20 分割，上传时选择"按比例分割 0.8/0.2"即可。

### 3.4 训练参数（火山方舟控制台配置）

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 训练类型 | SFT 精调 | 有监督微调 |
| 基座模型 | Doubao-Seed-1.6-flash | 轻量、快速 |
| 学习率 | 2e-5（默认） | 新手用默认值 |
| 训练轮数 (Epoch) | 3 | 防止过拟合 |
| 数据分割 | 0.8 / 0.2 | 训练/验证比例 |

### 3.5 操作步骤

1. **登录**：进入 [火山方舟控制台](https://console.volcengine.com/ark) → 模型精调
2. **上传数据**：
   - 点击「创建精调任务」
   - 选择「SFT 精调」
   - 上传生成的 `sft-train.jsonl` 文件（< 100MB 直接拖拽上传）
   - 数据分割选"按比例分割"，比例 0.8
3. **选择基座**：Doubao-Seed-1.6-flash
4. **配置参数**：学习率 2e-5，Epoch 3，其他保持默认
5. **提交训练**：预估费用 ¥50-100，训练时长约 1-2 小时
6. **评估**：训练完成后检查 Loss 曲线是否收敛

## 四、阶段 2：DPO 直接偏好优化

SFT 完成后，如果效果未达预期，进行 DPO 对齐。

### 4.1 数据策略

DPO 需要 `(prompt, chosen, rejected)` 三元组：

- **prompt**：待改写的 AI 风格文本
- **chosen**：改写后 AIGC 检测分数更低的版本（更"人类"）
- **rejected**：改写后 AIGC 检测分数较高的版本（仍然"AI味"）

#### 构造方法

1. 取 AI 风格文本作为 prompt
2. 用 SFT 微调后的模型生成 2-3 个改写候选
3. 用本地困惑度检测器（`perplexity-service.ts`）对每个候选计算 AI 评分
4. 评分最低的作为 chosen，评分最高的作为 rejected

### 4.2 数据格式（火山方舟 JSONL）

```jsonl
{"messages":[{"role":"system","content":"你是专业的文本润色编辑。将AI风格的文本改写为自然的人类写作风格。"},{"role":"user","content":"[AI风格文本]"}],"chosen":"[检测分低的改写]","rejected":"[检测分高的改写]"}
```

### 4.3 数据规模

| 数据集 | 条数 |
|-------|------|
| DPO 训练集 | 2000-3000 条 |
| DPO 验证集 | 300-500 条 |

### 4.4 训练参数

| 参数 | 推荐值 |
|------|--------|
| 训练类型 | DPO |
| 基座模型 | 阶段 1 的 SFT 产出模型 |
| 学习率 | 1e-5（DPO 需更低） |
| 训练轮数 | 2 |
| Beta | 0.1（默认） |

### 5.1 模型部署

1. 进入「模型产出」，选择最优的 checkpoint
2. 创建「在线推理接入点」

- `runAigcRewrite()` 的 `modelService.chat()` 调用切换为火山方舟 API
- 保留现有的词表替换和节奏扰动后处理
- 保留困惑度复检的候选选择机制

### 5.3 配置项

在 AI 服务设置中新增：
- 改写模型 API 地址（火山方舟 Endpoint）
- 改写模型 API Key
- 是否使用微调模型（开关）

## 六、数据生成脚本说明

### 脚本文件

| 脚本 | 用途 | 运行方式 |
|------|------|---------|
| `scripts/gen-sft-dataset.mjs` | 生成 SFT 训练数据 | `node scripts/gen-sft-dataset.mjs` |
| `scripts/gen-dpo-dataset.mjs` | 生成 DPO 训练数据 | 阶段 2 使用 |

### SFT 数据生成流程

```
人类小说文本 → 随机采样片段 → [方案A] 直接配对（人类文本即target）
                            → [方案B] LLM AI化 → (AI版本, 人类版本) 配对
```

**方案 A（纯规则，无需 LLM）**：
- 从人类文本采样作为 assistant 回答
- 用规则将人类文本"AI化"（替换口语为书面语、均匀化句长、插入模板连接词）作为 user 输入
- 优点：零成本、可大量生成
- 适合：快速产出初版数据集

**方案 B（LLM 辅助）**：
- 调用 LLM 将人类文本改写为 AI 风格
- 更真实但有 API 成本
- 适合：后续精调数据集质量

脚本默认使用方案 A，可通过 `--use-llm` 切换到方案 B。
