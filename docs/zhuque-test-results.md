## 基本规则

- **最低字数要求**: 350字
- **检测是确定性的**: 同一文本多次提交结果完全一致
- **输出三分类**: 人工特征 / 疑似AI / AI特征，三者之和为100%

## 八、朱雀检测模型推断

### 检测架构（推断）

```
输入文本
  ↓
分段（约200-300字一段）
  ↓
每段独立评分
  ├── Token级: 词汇概率分布 → AI/疑似/人工 基础分
  ├── 短语级: 匹配已知AI模式 → 加权调整
  │    ├── 电影镜头链 (权重极高)
  │    ├── 书面连接词 (权重高)
  │    ├── 模板情感/总结句 (权重中)
  │    └── 比喻/微动作/句长 (不参与)
  └── 分段分数
  ↓
加权汇总（首段权重偏高）
  ↓
三分类输出: 人工% / 疑似% / AI%
```

### 三个检测层次

| 层次 | 检测内容 | 效果 |
|------|---------|------|
| **Token层** | 词汇概率分布（是否为高频预测词） | 决定AI vs 疑似的基线 |
| **短语层** | 特定AI模式匹配 | 在基线上叠加惩罚 |
| **结构层** | 分段评分+位置加权 | 首段权重高 |

### 对应策略

| 检测层 | 最佳应对策略 | 效果 |
|--------|------------|------|
| Token层 | **低频词替换** | 100%AI → 100%疑似 |
| 短语层 | **删除电影镜头链 + 连接词** | 人类文本不被误判 |
| 结构层 | **人类文本前置 + 交替混合** | 提升人工分、消灭AI标签 |

## 十一、词级打乱梯度实验（WS 系列）

### 背景

对 `docs/mix-ai/M.md`（猫爬架短篇）做**段内全 token 打乱**后，朱雀给出 **人工特征 100%**——与原文（AI 文本）完全相反。

**原因推断**（与第八节 Token 层一致）：

| 打乱方式 | 对 token 分布的影响 | 对语义的影响 | 朱雀预期 |
|---------|-------------------|------------|---------|
| **WS1 全 token** | n-gram 断裂，相邻词概率极低 → 像「随机噪声」 | 完全不可读 | **→100% 人工**（误判） |
| **WS2 句内分句** | 逗号边界内词序不变，仅分句顺序变 | 基本可读 | 应接近原文 |
| **WS3 短语块** | 短语内部不变，仅块顺序变 | 可读，略拗口 | 应接近原文 |
| **WS4 相邻交换** | 仅 swap 32% 相邻词对 | 几乎不变 | 应几乎等同原文 |

**结论**：朱雀**不检测「词序是否合理」**，而是检测 **token 共现概率**。全打乱把高概率 n-gram 拆散，反而触发「非 AI 预测区」→ 人工分拉满。

### 优化策略（语义保留词级打乱）

已实现 `scripts/word-shuffle.mjs`，四种模式：

```bash
# 单文件
node scripts/word-shuffle.mjs --mode phrase docs/experiments/A2-ai.txt out.txt

# 批量生成 WS 实验（22 个基线 × 4 模式 = 88 文件）
node scripts/gen-word-shuffle-experiments.mjs
node scripts/gen-word-shuffle-experiments.mjs --all   # 全部 experiments/*.txt
```

| 模式 | 说明 | 语义 | 用途 |
|------|------|------|------|
| `full` | 段内 token + 标点全打乱 | ❌ | 对照：验证「乱序=人工」假说 |
| `clause` | 句内按逗号分句打乱，句序不变 | ✅ | 测试分句顺序是否影响检测 |
| `phrase` | 句内分词短语块打乱（助词附着前词） | ✅ | **推荐**：词级扰动且可读 |
| `swap` | 句内 32% 相邻词对交换 | ✅✅ | 测试极小扰动是否影响检测 |

`docs/mix-ai/M.md` 已用 **`phrase` 模式**（seed=42）重新生成。

生成文件命名：`WS{1-4}-{mode}-{原文件名}.txt`，详见 `docs/experiments/WS-manifest.json`。

### 实验结论

**全部 12 项测试完成。**

#### 核心发现

1. **WS1（全token打乱）、WS3（短语块打乱）、WS4（32%相邻交换）→ 全部 100% 人工**
   - 无论原文是纯 AI（A2=100%AI）、疑似（M4=100%疑似）、还是混合（Q6=57%人工），只要做了**词级扰动**，朱雀一律判 100% 人工
   - 甚至 WS4 仅交换 32% 的相邻词对，扰动极小，依然 100% 人工

2. **WS2（分句打乱）→ 唯一不到 100% 的模式**（A2: 0/63/37）
   - 分句打乱**不改变分句内部的词序**，仅重排逗号分隔的短句顺序
   - 分句内部 n-gram 完整保留 → 朱雀仍能部分识别
   - 但比原文（100%AI）已显著下降，AI 从 100% 降至 37%

3. **人类文本打乱后不受影响**（A1: 打乱前后均为 100% 人工）
   - 词级打乱**只降 AI 分，不误伤人类文本**

#### 朱雀检测粒度定位

```
朱雀检测粒度 = 相邻 2-3 词的 n-gram 共现概率

分句打乱（WS2）：保留分句内 n-gram → 部分检出（63% 疑似 + 37% AI）
词级打乱（WS3/WS4）：破坏 n-gram → 100% 人工（哪怕只动 32% 的词对）
```

| 操作 | n-gram 是否断裂 | 朱雀结果 |
|------|---------------|---------|
| 原文不动 | ❌ 完整 | 按原文判定 |
| WS2 分句重排 | ❌ 分句内完整 | 部分降级（AI→疑似） |
| WS4 交换 32% 词对 | ✅ 局部断裂 | **100% 人工** |
| WS3 短语块打乱 | ✅ 大量断裂 | **100% 人工** |
| WS1 全 token 打乱 | ✅ 全部断裂 | **100% 人工** |

#### 对改写策略的启示

1. **词级打乱是朱雀的致命弱点** — 任何 n-gram 级扰动都能完全规避检测
2. **但打乱后文本不可读** — 不能直接用于实际输出，只能用于：
   - 验证朱雀检测机制（已完成）
   - 作为改写策略的理论依据
3. **实际对抗仍应使用**：
   - **DeepSeek 改写**（天然低检出 + 可读）→ 第七节
   - **低频词替换**（不破坏语序但改变 token 分布）→ 第四节
   - **人类文本交替混合**（利用结构层权重）→ 第五节
4. ~~新的可探索方向：可读的 n-gram 级扰动~~ → **已测试，见第十二节**

## 十二、可读 n-gram 断裂实验（RB 系列）

### 背景

WS 系列证明词级打乱可 100% 骗过朱雀，但文本不可读。本节测试四种**保持可读性**的 n-gram 断裂策略。

### 策略设计

| 策略 | 做法 | 可读性 | 预期 |
|------|------|--------|------|
| **RB1** 虚词插入 | 动词后加「了/着/过」(~20%)，逗号前加「呢/吧」(~6%) | ★★★★ | 插入 token 打断 bigram |
| **RB2** 同义替换 | 60+ 常见词→同义词（看着→瞧着，里面→里头），55% 命中率 | ★★★★★ | 替换 bigram 中的词 |
| **RB3** 句法重构 | 分句前后互换(20%)、把字句转换(35%)、状语后移(30%) | ★★★★ | 重排改变长程 n-gram |
| **RB4** 组合 | RB3 → RB2 → RB1 三层叠加 | ★★★ | 多层叠加 |

### 测试结果（A2 纯 AI 文本）

| 策略 | 人工特征 | 疑似AI | AI特征 | 原文 |
|------|---------|--------|--------|------|
| RB1 虚词插入 | 0% | 0% | **100%** | 100% AI |
| RB2 同义替换 | 0% | 0% | **100%** | 100% AI |
| RB3 句法重构 | 0% | 0% | **100%** | 100% AI |
| RB4 三者组合 | 0% | 0% | **100%** | 100% AI |

**四种策略全部失败。**

### 关键分析：为什么 WS4 能过而 RB 不能？

| 对比 | WS4（32%词对交换） | RB4（三层组合） |
|------|-------------------|----------------|
| **影响范围** | 32% 的**所有**相邻词对 | 仅 5-15% 的文本被修改 |
| **n-gram 断裂密度** | **每 3 个 bigram 就有 1 个断裂** | 每 10-20 个 bigram 才有 1 个改变 |
| **断裂分布** | **均匀分布在全文** | 集中在少数命中点 |
| **结果** | 100% 人工 | 100% AI |

### 修正后的朱雀机制理解

之前结论「朱雀检测粒度 = 相邻 2-3 词 n-gram」需要**补充**：

```
朱雀检测的不是「是否存在 AI n-gram」
而是「全文 n-gram 的聚合概率分布」

- 修改 5-15% 的 token → 聚合分布几乎不变 → 仍判 AI
- 打乱 32%+ 的词对 → 聚合分布根本性改变 → 判人工
- 阈值在「影响 >30% 相邻词对」附近
```

**类比**：就像检测一杯水是淡水还是海水——
- 加几滴淡水（RB 策略）→ 仍是海水
- 把水搅浑打散（WS 打乱）→ 检测仪读数完全变了
- 换一杯水（DeepSeek 改写）→ 从源头改变

### 结论

**可读的 n-gram 局部扰动无法骗过朱雀。** 原因：

1. **虚词（了/着/呢/吧）** 本身就是 AI 高概率预测的 token，插入后不改变分布
2. **同义替换**密度不足（~5-10%），且替换词（如「瞧着」替代「看着」）仍可能在 AI 预测的 Top-K 区间内
3. **句法重构**只影响极少数句子，全文 n-gram 分布几乎不变

### 可行路线（经验证）

| 方法 | 可读性 | 效果 | 原理 |
|------|--------|------|------|
| **DeepSeek 改写** | ★★★★★ | 100%人工(+50%人类混合) | 从源头重生成，天然低概率分布 |
| **H1 低频词替换** | ★★★★ | 100%AI → 100%疑似 | 针对性替换高概率 token |
| **人类文本交替混合** | ★★★★★ | 结构层权重利用 | 分段评分 + 位置加权 |
| ~~可读 n-gram 扰动~~ | ★★★★★ | ❌ 无效 | 密度不足以改变聚合分布 |

## 十三、PPL 引导实验（PPL/SR 系列）

### 背景

前面的实验证明两个事实：
1. 可读的词级替换无法骗过朱雀（RB 系列全部失败）
2. DeepSeek 改写可以消灭 AI 标签（D7: 100%AI → 100%疑似）

本节测试两种 PPL 引导策略：
- **PPL 同义替换**：用本地模型找出高概率词，替换为同义词
- **SR 选择性改写**：用 PPL 定位 AI 浓度最高的句子，只改写那些句子

### 实验一：PPL 引导同义替换

用 Qwen-4B 计算每个 token 的预测概率，按概率降序选择可替换词，用同义词替换。

| 策略 | 人工特征 | 疑似AI | AI特征 | 替换量 |
|------|---------|--------|--------|--------|
| F1 原文 | 0% | 85.03% | 14.97% | 0 词 |
| PPL100（全部可替换词） | 0% | 56.01% | **43.99%** | 27 词 |

**结果：AI 特征反而从 15% 涨到 44%！**

**分析**：同义替换本身就是 AI 改写的典型手法。朱雀检测的不只是 token 概率分布，还包括「替换痕迹」——用"声儿"替代"声音"、用"物件"替代"东西"恰恰是 AI 改写的指纹。

### 实验二：PPL 引导选择性 DeepSeek 改写（SR 系列）

用 Qwen-4B 逐句计算 avgProb（平均 token 预测概率），按 AI 浓度排序，只改写 Top-K% 的高浓度句。

**脚本**: `scripts/ppl-selective-rewrite.mjs`

**F1-ai-novel.txt 逐句 PPL 评分摘要**：
- 全文 102 句（排除短于 4 字的），平均概率 13.5%
- 高浓度区（avgProb >20%）：31 句，多为叙述性长句
- 低浓度区（avgProb <6%）：短对话/感叹词，模型几乎无法预测

| 策略 | 人工特征 | 疑似AI | AI特征 | 改写量 |
|------|---------|--------|--------|--------|
| F1 原文 | 0% | 85.03% | **14.97%** | 0 句 |
| **SR30**（Top 30%） | 0% | **100%** | **0%** | 31 句 |
| **SR50**（Top 50%） | 0% | **100%** | **0%** | 51 句 |
| SR70（Top 70%） | 0% | 84.68% | 15.32% | 72 句 |
| SR100（全文） | 0% | 85.95% | 14.05% | 102 句 |

### 关键发现：选择性改写优于全文改写

```
SR30/SR50（只改高浓度句）→ AI特征归零 ✓
SR70/SR100（改写大部分/全文）→ AI特征回到 ~15% ✗
```

**原因分析**：

| 因素 | SR30（成功） | SR100（失败） |
|------|------------|-------------|
| 高浓度句 | DeepSeek 改写 → 概率下降 | DeepSeek 改写 → 概率下降 |
| 低浓度句（对话/碎片） | **保留原样 → 天然低概率锚点** | DeepSeek 改写 → 概率反而升高 |
| 整体分布 | 高概率峰被削 + 低概率锚保留 | 全部被 DeepSeek 统一 → 新的均匀分布 |

短对话句（"走了""门关上""大二"）天然 avgProb ≈ 0%，是朱雀眼中的「人工锚点」。
全文改写把这些锚点也重写了，使其变得可预测，反而拉高了整体 AI 分布。

### 结论

1. **PPL 引导有价值**——不是为了效率，而是为了效果。选择性改写比全文改写更好
2. **最优策略：只改写 AI 浓度最高的 30-50% 句子**，保留其余天然低概率部分
3. **同义替换方向彻底否定**——不论是否有 PPL 引导，词级同义替换都会加重 AI 检测
4. SR30 可消灭 AI 标签（100%疑似），但仍无法突破到「人工特征」——需要配合人类文本混合或词对交换（见第十四节）

## 十五、可读性优化实验（SA/SB/SC 系列）

### 背景

V3（100%人工）的 50% 词对交换导致文本基本不可读（人名拆散、量词错位、句式破坏）。本节测试三种策略在保持人工特征的前提下提升可读性。

### 策略 A：句级独立改写 + 轻度 swap 叠加

每个句子单独发给 DeepSeek 改写（制造跨句 n-gram 断裂），然后叠加轻度 swap。

| 策略 | 做法 | 人工特征 | 疑似AI | AI特征 |
|------|------|---------|--------|--------|
| SA2 | 句级独立改写 + 20% swap | 0% | **100%** | 0% |

**失败**：跨句 n-gram 断裂 (~10%) + 20% swap 的总扰动不足以穿越阈值。

### 策略 B：受保护智能 swap

保护人名/量词/固定搭配不被交换，对其余词对以更高交换率执行 swap。

| 策略 | 交换率 | 人工特征 | 疑似AI | AI特征 |
|------|--------|---------|--------|--------|
| SB1 | 55% | **13.61%** | 86.39% | 0% |
| SB2 | 65% | **28.75%** | 71.25% | 0% |
| SB3 | 80% | 0% | **100%** | 0% |

**发现**：SB2 (65%) 最优，但 SB3 (80%) 反而退回 100%疑似。保护词汇在高交换率下成为可预测的「锚定岛」，反而帮助朱雀识别。

### 策略 C：V3 后修复（DeepSeek 最小限度修复）

从 V3（100%人工）出发，让 DeepSeek 做最小限度的词序修复。

| 策略 | 做法 | 人工特征 | 疑似AI | AI特征 |
|------|------|---------|--------|--------|
| SC1 | 每句最多修复 3 对词序 | 0% | **100%** | 0% |

**失败**：DeepSeek 的修复本身引入了可预测的 AI n-gram 模式，抵消了原有的扰动。

### 核心结论

**LLM 参与 = 引入可预测 n-gram = 失败**

```
路径1：swap 制造混乱 → LLM 修复可读性 → LLM 重新引入 AI 模式 → 疑似AI
路径2：LLM 独立改写 → swap 叠加扰动 → 总扰动不足 → 疑似AI
路径3：保护关键词 + 高 swap → 保护词成为检测锚点 → 疑似AI
```

**可读性与反检测的根本矛盾**：
- 可读性要求词序合理 → 合理词序 = 可预测 n-gram → 被检测
- 反检测要求 n-gram 不可预测 → 不可预测词序 = 不可读
- 唯一打破这个矛盾的是**真实人类文本**，因为人类写作天然包含不可预测但可读的词序选择

## XVIII. RE9–RE18：可读性突破（保护虚词 swap）

### 问题

**RE7**（52% 全词 swap + 仅人名修复）= **84.86% 人工**，但叙述段可读性仍差：`城同快递`、`三给猫换只水`、`被像穴点了` 等。根因：**相邻 swap 把「的/了/把/在」等虚词也打乱**，语法崩坏远大于实词 swap 对朱雀的扰动价值。

### 新策略

1. **保护虚词 swap（RE13–RE16）**：仅交换两个 token 均 ≥2 字且非虚词/结构词的相邻对；虚词位置固定，可读性接近 SR30。
2. **分层规则修复（RE9–RE12）**：在 52% 全词 swap 上叠加量词/关键句式修复（RE10 tier2）。
3. **扩展规则 + 更高 swap（RE17–RE18）**：54–56% swap + GLOBAL_FIXES 前 80 条。

**RE14 结论（实测）**：可读性 ★★★★☆，但朱雀 **100% 疑似AI、0% 人工特征** — 与 RE7（84.86% 人工）形成极端对立。

**根因（实测有效 swap 率）**：

宽保护 + tier2 规则修复 = 语法接近标准中文 + 几乎无 bigram 扰动 → **100% 疑似AI**。

## XIX. 三角 trade-off（更新）

**核心机制（RE19–22 实测确认）**：朱雀检测的是 **虚词相邻 bigram 聚合分布**，不是 swap 次数。保留「了/的/在/把」位置 → 无论实词 swap 多少 → 100% 疑似AI。

```
        可读性
          ▲
          │  RE14 / RE19–22（虚词位置正确）

          │
   RE25 ─── 虚词靶向 → 100%疑似AI ✗
   RE27 ─── 双 pass 26%全swap → 100%疑似AI ✗
          │
          │  RE7（52%全swap+人名）→ 84.86% 人工 ✓ 唯一有效
          ▼
        人工特征
```

**RE 系列收官（RE27 实测）**：无人类文本 + 可读 + 高人工分 **不可兼得**；采用 **RE7** 或 **混人类文本**。
