# 小说创作辅助系统 - V1.0 开发任务清单

> **版本目标**：V1.0 基础创作版本 — 完成 Electron 桌面应用骨架、SQLite 数据库、文风模块、核心创作流程（大岗孵化器→正文生成）、锚点注入、过程干预、模型对接、灵感板、作品管理。

> **技术栈**：Electron + Vue3 + TypeScript + Tailwind CSS + SQLite (better-sqlite3) + Tiptap  
> **来源文档**：`小说创作辅助系统-完整需求与开发方案.md` v2.1

## Phase 6：核心创作流程

### 6.1 步骤导航与状态管理
- [ ] 实现 6 步进度条组件：初始想法 → 核心设定 → 分卷大纲 → 分章情节 → 章节大纲 → 正文生成
- [ ] 每步状态：`pending` / `generating` / `done` / `rejected`
- [ ] 支持步骤跳转、回退、重新生成
- [ ] 每步生成结果持久化存储

### 6.2 大岗孵化器（创作前置）
- [ ] 大岗诊断：作者输入初步想法 → AI 输出核心吸引力/类似故事/差异化/潜在风险分析
- [ ] 大岗变体探索：按维度（性别反转/时代变更/冲突转换/视角切换）生成变体矩阵
- [ ] 大岗对标分析：作者指定对标作品 → AI 分析核心要素并融合建议
- [ ] 关键场景锚点收集：文本输入框收集作者不可妥协的核心画面
- [ ] 情感基调定调：提供情绪标签选择 + 自定义输入
- [ ] 倒推大纲法：锚定高潮/结局 → AI 反向推导故事骨架 → 生成"倒推大纲树"
- [ ] 锁链式嵌套预览：基于倒推结果，展示 A1-A2-B1-A3-B2-C1-B3 嵌套结构
- [ ] 输出产物：扩写方向 + 锚点清单 + 情感基调声明 + 倒推大纲树 + 嵌套结构预览

### 6.3 步骤1：初始想法扩写
- [ ] AI 生成 3-5 个不同方向的扩写版本
- [ ] 每个版本展示：核心亮点、核心冲突、故事基调、受众定位、锚点对齐度评分
- [ ] 作者选择版本 + 手动修改 + 微指令调整
- [ ] AI 编辑模式自评：独特性评分、对标作品重合度

### 6.4 步骤2：核心设定生成
- [ ] 结构化输出：人物设定（姓名/年龄/身份/性格/口头禅/内在矛盾/行为逻辑/关系图谱）、世界观设定、核心冲突设定
- [ ] 支持按设定类别单独修改
- [ ] 角色功能定位规则引擎（见 Phase 9）

### 6.5 步骤3：分卷大纲生成
- [ ] 每卷包含：卷名、核心主题、核心冲突、起止时间点、叙事结构节点、主要事件节点、卷结尾钩子
- [ ] 支持调整分卷数、修改卷内容、调整顺序

### 6.6 步骤4：单卷分章情节生成
- [ ] 将一卷拆分为若干章的情节概要
- [ ] 每章包含：核心情节、起承转合节点、与前后章的因果链

### 6.7 步骤5：章节大纲生成
- [ ] 单章详细大纲：场景划分、对话节点、描写重点、情感走向
- [ ] 支持锚点绑定标记

### 6.8 步骤6：正文生成
- [ ] 基于章节大纲 + 全部锚点 + 文风约束，AI 生成完整正文
- [ ] Tiptap 富文本编辑器展示，支持作者手动修改
- [ ] 字数统计
- [ ] 生成后锚点对齐检测（见 Phase 7.3）

**验收标准**：完整走通 大岗孵化器 → 步骤1~6 → 正文生成 的全流程。

## Phase 8：过程干预通道

### 8.1 微指令注入
- [ ] 在任意生成结果旁提供"微指令"输入框
- [ ] 输入一句话调整指令 → AI 局部重写（而非全部重来）
- [ ] 保留修改历史（微指令 + 修改前后对比）

### 8.2 条件式预设
- [ ] 全局规则配置页：输入自然语言规则（如"任何角色死亡必须暂停""吻戏跳过""每章情节节点不超过5个"）
- [ ] 每次生成前自动将规则列表注入 Prompt
- [ ] 规则触发检测：生成内容命中某条规则 → 自动暂停并提示作者

### 8.3 A/B 流生成
- [ ] 在关键分叉点，作者点击"A/B 探索"
- [ ] AI 同时生成 A/B 两个版本（约 500 字）
- [ ] 作者选择版本后 AI 继续基于选中版本展开

### 8.4 否决即学习
- [ ] 作者否决生成结果时，弹出否决原因标签选择：太俗套/不符合角色/节奏太慢/对话不自然/偏离锚点
- [ ] AI 按原因标签重新生成
- [ ] 否决记录写入 `generation_log`（author_action=rejected, reject_reason=标签）

**验收标准**：可随时通过微指令调整生成方向；条件预设自动拦截；A/B 流可在分叉点选择；否决后自动带原因重试。

## Phase 9：角色功能定位规则引擎

### 9.1 规则定义
- [ ] 主角功能性 > 完美性：检测主角设定是否包含明确的故事功能（推动剧情、制造冲突、承担主题），而非仅堆砌完美标签
- [ ] 配角记忆点 > 完整性：检测配角是否具备独特的记忆标签（外貌/行为/语言/时间），而非追求完整背景故事
- [ ] 反派破坏力 > 邪恶值：检测反派的破坏能力是否具体、可信、对主角构成真实威胁
- [ ] 群像独立诉求 > 服务主角：检测次要角色群是否具备独立动机，而非仅为主角服务

### 9.2 检查引擎
- [ ] 在核心设定生成后，自动运行角色功能定位检查
- [ ] 输出检查报告：每个角色的功能定位评分 + 改进建议
- [ ] 检查结果以卡片形式展示在步骤2结果面板

**验收标准**：核心设定生成后自动输出角色功能定位检查报告。
